# 地圖到底有沒有用？12-cell A/B 實測報告（Map-Value Evaluation）

> 問題只有一個：**「給 AI 稜鏡地圖包，是否優於直接給原料？」**（答對率／token／導向速度／幻覺）
> 語料為私有專案文件與程式庫，**不隨 repo 散布**；本檔記錄設計、全部數據與失敗模式。
> 測試執行與評分：Claude Code（2026-07-18）。評分依答案鑰逐條可覆核，但非盲評（見「誠實限制」）。

## TL;DR

- **地圖買到的是「答對率與出處正確性」，不是省 token**——強代理模型在大語料上 pack 8/8 vs raw 6.5/8，但 token 反多 24%。
- **語料小到讀得完時，地圖是純開銷**。
- **單發小模型上地圖無一致優勢**——瓶頸是 ctx 預算與調閱順序，不是找不到檔。
- **防幻覺的主變因是模型紀律＋「語料裡沒有就明說」的 prompt 約束**，不是地圖。

## 設計

- **2 語料域**：CODE＝私有 Python 桌面應用 repo 子集（45 檔，416KB）；DOC＝私有中文工作報告（4 檔，82KB）
- **3 模型**：Claude Haiku（雲端代理型，可用檔案/搜尋工具）；Gemma 4 E4B、Qwen2.5-7B（本機 Ollama，單發）
- **2 條件**：**pack**＝稜鏡地圖包（summary-only 地圖＋依地圖調閱原檔）vs **raw**＝直接給原料
- **8 題/域**：6 真題（導航/理解/深導航）＋ **2 陷阱題**（答案不在語料＝幻覺負對照）
- 公平性：同資訊預算（16,000 字元）、temp 0、每 cell 獨立工作區、答案鑰與受測隔離
- 本機協議：pack＝兩回合（T1 看地圖點檔 → T2 餵所點檔）；raw＝單回合塞滿預算後截斷
- 評分制：每題 1 分（部分 0.5）；陷阱題明說「語料中沒有」＝1、捏造＝0

## 五條發現

1. **地圖的實證價值＝強代理模型 × 大語料下的答對率與出處正確性。** Haiku code pack 8/8，45 檔只精開 9 檔且全是含答案檔；raw 用 grep 自導航省了 token，但「搜到就信」→ 出處誤指 ×2＋單位誤標 ×1。**地圖省下的是錯誤，不是 token**（pack 反多 24%）。
2. **語料讀得完＝地圖純開銷。** doc 域（4 檔）兩組同滿分；pack 組拿了地圖照樣全檔通讀（+3.2k tokens 零效益）。
3. **單發小模型上地圖無一致優勢。** E4B doc +1.5／code **−2**：地圖點對了檔，但 fetch 順序讓第一個大檔吃光預算，其餘檔餓死；raw 組反而因摘要類檔案（CHANGELOG 等）恰好排前而得分。要讓小模型受益，地圖需要「摘要即答案」密度或聰明調閱（小檔優先／區段級）。
4. **「省 90% token」類宣稱在「還要答對」的前提下不成立。** 成立的只是「索引比全文小」——把索引當提示詞的一部分後，pack 在所有本機 cell 都比 raw 貴 29%–100%（地圖本體＋兩回合都是成本）。本 repo 的 token 數字一律指「包體積 vs 原文體積」，不隱含端到端問答成本更低。
5. **防幻覺的主變因是模型紀律，不是地圖。** 相同 prompt 約束（「語料裡沒有就明說，禁用背景知識補」）下：Gemma 系（Haiku＋E4B）20 次陷阱全守、真題零捏造；Qwen2.5-7B 四格共 9 處捏造＋唯一一次陷阱失守（為不存在的 CLI 指令發明參數）。地圖救不了紀律——Qwen 捏造最多的一格正是 pack 組。

## 實用建議

- **用地圖**：代理型模型＋語料一次讀不完（數十檔以上）＋答案的「出處」重要時。
- **跳過地圖**：語料 < 模型可通讀規模——直接餵，別付地圖稅。
- **單發本機小模型**：先解決調閱策略（預算內小檔優先、區段級 fetch），再談地圖。
- **任何 RAG／委外問答任務**：prompt 必帶「語料裡沒有就明說」——12 cells 中它是比任何工程手段都穩的防幻覺閥（16/16 次守住的 cell 都帶著它）。

## 誠實限制

- **本報告屬探索性單樣本實驗**（2026-07-18 三席外部評審後加註）：發現 1／2／5 有跨 cell 一致趨勢；**發現 3（單發小模型）與 4（token 宣稱）受 fetch 順序與切片偶發性影響較大，外推前需 n≥5 重複驗證**（已列後續計畫）。
- 每 cell n=1（無重複取樣）；題庫 n=8/域；模型比較（尤其 Qwen 紀律觀察）是單次觀察，非統計結論。
- 出題者＝評分者＝實驗設計者；評分依答案鑰逐條可覆核，但非盲評。
- Haiku（代理式多輪）與本機模型（兩回合固定協議）不同構，跨模型只看趨勢不看絕對值。
- 代理環境的系統上下文含一行與某陷阱題相關的資訊（部分洩題）；四個 cell 均正確以「語料為準」排除，未影響評分，據實記錄。
- 私有語料不公開，故第三方無法逐字重跑；題型範本（6 真題＋2 陷阱、負對照設計）可直接套用到你自己的語料複現。

## 複現方式（用你自己的語料）

```bash
# 1. 打包你的語料
python prism/prism_pack.py <你的語料夾> -o pack/

# 2. 出 8 題：6 題答案確定在語料內、2 題確定不在（陷阱＝負對照）
#    ⚠ 出題人也要負對照：每題先全文檢索確認「在／不在」，我們差點把真功能當陷阱

# 3. 同模型同預算跑 pack vs raw 兩條件，陷阱題「明說沒有」計 1 分、捏造計 0 分
```
